### 1. Постановка задачи
Разработать программу для работы с расписанием автобусов на остановке. Реализовать
функционал добавления и удаления автобусов, управления временем их прибытия, а
также обработки запросов, связанных с расписанием.
Описание функционала
1. Вывод расписания
  Отображение расписания в формате:

    НомерАвтобуса1: hh:mm, hh:mm

    НомерАвтобуса2: hh:mm, hh:mm

    Все моменты времени в расписании упорядочены хронологически.
  
2. Добавление времени прихода автобуса
  Добавление времени прибытия автобуса по его номеру и времени в формате «hh:mm».
  Если автобуса с данным номером ещё нет, он добавляется в расписание. Если указанное время уже есть, оно не добавляется.

4. Удаление автобуса из расписания
  Удаление автобуса с указанным номером из расписания.

5. Удаление времени прихода автобуса
  Удаление конкретного времени прибытия автобуса по его номеру и указанному
времени.

6. Добавление автобуса с периодическими остановками (по количеству)
  Добавление автобуса, который начинает ходить с указанного времени и делает
остановки через заданный интервал (в минутах). Указывается номер автобуса,
время начала, интервал и количество остановок.

7. Добавление автобуса с периодическими остановками (по времени)
  Добавление автобуса, который начинает ходить с указанного времени и делает
остановки через заданный интервал до указанного времени включительно. Указывается номер автобуса, время начала, интервал и конечное время.

8. Ожидание автобуса человеком
  Человек приходит на остановку в указанное время и ждёт автобус с переданным
номером определённое количество минут. Определяет, дождётся ли человек автобуса.

9. Автобусы на отрезке времени (в пределах суток)
  Возвращает номера автобусов, которые приезжают на остановку в указанный отрезок времени (без повторений). Гарантируется, что отрезок времени находится в
пределах одних суток.

10. Автобусы на отрезке времени (через полночь)
  Возвращает номера автобусов, которые приезжают на остановку в указанный отрезок времени, включая переход через полночь (без повторений).

11. Первый автобус для человека
  Человек приходит на остановку в указанное время и садится на первый автобус
из переданного списка номеров. Возвращает номер автобуса, который приедет
первым


### 2. Описание классов

  #### Класс Time
  В нем хранится время в двух переменных типа `int` - количество часов и минут. Также в нем есть следующие методы:
  
  - Конструктор от строки и от двух переменных типа `int`
  - Проверка на равенство объектов класса Time (от переменной класса Time)
  - Строгое и нестрогое сравнение двух объектов класса Time (от двух переменных класса Time). Статические методы
  - Проверка, находится ли время в указанном интервале (от двух переменных класса Time)
  - Сложение времени (от двух переменных класса Time). Статический метод

  #### Класс Bus
  В нем хранится номер автобуса в переменной типа `int` и массив времени класса Time длины 1440 - количество минут в сутках. В нем есть следующие методы:

  - Конструктор от переменной типа `int` - номера, и строки или переменной типа Time
  - Переопределение метода toString для класса Bus
  - Добавление и удаление времени автобуса
  - Нахождение наименьшего времени автобуса, большего данного
  - Проверка, если у автобуса данное время
  - Проверка, приедет ли автобус в указанный промежуток времени

  #### Класс BusStop
  В нем хранится массив автобусов класса Bus длины 100 (число взято с потолка, но с использованием здравого смысла). В нем есть следующие методы:

  0. Конструктор
  1. Показ расписания
  2. Добавление времени автобуса
  3. Удаление автобуса
  4. Удаление конкретного времени у конкретного автобуса
  5. Добавление автобуса, приезжающего через заданный интервал заданное количество раз
  6. Добавление автобуса, приезжающего через заданный интервал до данного времени окончания
  7. Проверка, дождется ли пассажир, пришедший в данное время, данного автобуса, если он будет ждать данное количество минут
  8. Показ автобусов, приезжающих в некотором периоде в течение одних суток
  9. Показ автобусов, приезжающих в некотором периоде
  10. Показ первого приезжающего автобуса из указанного списка

  А также "рабочие" методы
  
  - Проверка, есть ли в переданном списке номеров, данный
  - Добавление автобуса по уже инициализированному времени
  - Проверка, есть ли автобус с таким номером
